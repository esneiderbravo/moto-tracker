# Contributing to MotoTracker 🏍️

To maintain a high-quality codebase and a clean history, please follow these guidelines:

## 🌿 Branch Naming Convention

All new features or fixes must be developed in a separate branch following this pattern:
`feature/key-title-of-features-implemented`

Example: `feature/ai-maintenance-suggestions`

## 🔄 Workflow

1. **No Direct Pushes to `main`:** All changes must go through a Pull Request.
2. **Create a Branch:** Start from `main` and create your feature branch.
3. **Commit often:** Use descriptive commit messages (e.g., `feat: ...`, `fix: ...`, `docs: ...`).
4. **Open a PR:** Once ready, open a Pull Request towards `main`.
5. **Review:** Ensure the code passes all lints and follows the project's premium design standards.

## 🛠️ Code Standards

- **Mobile-First:** Always test UI changes on mobile view.
- **English Code:** All variables, functions, and comments must be in English.
- **Zero-Cost:** Prioritize using free-tier services and lightweight libraries.

Thank you for helping MotoTracker grow! 🚀
